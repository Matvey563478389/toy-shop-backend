# Toy Shop Backend

Backend API для интернет-магазина игрушек, построенный на Node.js и Express с использованием PostgreSQL.

## 🚀 Технологии

- **Node.js** - JavaScript runtime
- **Express.js** - веб-фреймворк
- **PostgreSQL** - база данных
- **JWT** - аутентификация пользователей
- **bcrypt** - хеширование паролей
- **multer** - загрузка файлов
- **node-pg-migrate** - миграции базы данных

## 📋 Функциональность

- **Аутентификация пользователей** - регистрация и вход с JWT токенами
- **Админ-панель** - управление товарами и пользователями
- **Управление игрушками** - CRUD операции для товаров
- **Загрузка изображений** - поддержка загрузки файлов для товаров
- **Ролевая система** - разграничение прав доступа (admin/user)

## 🛠️ Установка и запуск

### Предварительные требования

- Node.js (версии 16+)
- PostgreSQL
- npm или yarn

### 1. Клонирование репозитория

```bash
git clone https://github.com/Matvey563478389/toy-shop-backend.git
cd toy-shop-backend
```

### 2. Установка зависимостей

```bash
npm install
```

### 3. Настройка переменных окружения

Скопируйте файл примера и настройте переменные:

```bash
npm run replace-env
```

Отредактируйте `.env` файл с вашими настройками базы данных и JWT секретом.

### 4. Запуск БД И миграций

```bash
docker compose up -d --build
npm run migrate:up
```

### 5. Для заполнения тестовыми данными

```bash
npm run seed
```

### 6. Запуск приложения

Для разработки:
```bash
npm run dev
```

Для продакшена:
```bash
npm start
```

Сервер будет доступен на `http://localhost:3000`

## 🔗 API Эндпоинты

### Аутентификация
- `POST /api/auth/register` - Регистрация пользователя
- `POST /api/auth/login` - Вход пользователя

### Игрушки
- `GET /api/toys` - Получение списка игрушек
- `GET /api/toys/:id` - Получение игрушки по ID
- `POST /api/toys` - Создание новой игрушки (требует admin прав)
- `PUT /api/toys/:id` - Обновление игрушки (требует admin прав)
- `DELETE /api/toys/:id` - Удаление игрушки (требует admin прав)

### Пользователи
- `GET /api/users` - Получение списка пользователей (только для admin)
- `GET /api/users/:id` - Получение пользователя по ID

## 🔐 Аутентификация

API использует JWT токены для аутентификации. После входа пользователь получает токен, который должен передаваться в заголовке `Authorization: Bearer <token>`.
